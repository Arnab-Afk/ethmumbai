# web3deploy CLI

The official CLI for **D3PLOY** — deploy your site to IPFS and update your ENS contenthash in one command.

## Installation

```bash
# From this directory
npm link

# Or globally from npm (once published)
npm install -g web3deploy
```

## Quick start

```bash
# 1. Initialise a project
web3deploy init

# 2. Fill in .env with your keys (copied from .env.example)
cp .env.example .env

# 3. Deploy
web3deploy push
```

## Commands

| Command | Description |
|---|---|
| `web3deploy init` | Interactive setup — creates `web3deploy.config.json` + `.env.example` |
| `web3deploy push` | Build → upload to IPFS → update ENS contenthash |
| `web3deploy status [domain]` | Show current CID, IPNS entry, and recent deploys |
| `web3deploy rollback [id]` | Re-point ENS contenthash to a previous deploy |
| `web3deploy env set <key> <value>` | Store a value in ENS text records |
| `web3deploy env get <key>` | Read a value from ENS text records |
| `web3deploy env list` | List all ENS text records for your domain |

## Options

### `push`

| Flag | Default | Description |
|---|---|---|
| `-r, --repo <url>` | auto-detected from `git remote` | GitHub repo to deploy |
| `-d, --domain <ens>` | from `web3deploy.config.json` | ENS domain to update |
| `-e, --env <name>` | `production` | Deploy environment label |
| `--ipns-key <key>` | `IPNS_KEY` env var | IPNS key for custom domains |

### `env`

| Flag | Description |
|---|---|
| `-d, --domain <ens>` | Override the domain from config |

## Environment variables

| Variable | Description |
|---|---|
| `WEB3DEPLOY_API` | Backend URL (default: `http://localhost:3001`) |
| `WEB3DEPLOY_TOKEN` | Bearer token for authenticated API requests |
| `DEPLOYER_PRIVATE_KEY` | EOA private key for ENS transactions |
| `PINATA_JWT` | Pinata API JWT for IPFS pinning |
| `IPNS_KEY` | IPNS key bytes for custom domain updates |

## Config file — `web3deploy.config.json`

Generated by `web3deploy init`. Example:

```json
{
  "domain": "myapp.eth",
  "build": {
    "command": "npm run build",
    "outputDir": "dist",
    "framework": "vite"
  },
  "pinning": {
    "pinata": { "jwt": "${PINATA_JWT}" }
  },
  "signer": {
    "privateKey": "${DEPLOYER_PRIVATE_KEY}"
  },
  "apiUrl": "http://localhost:3001"
}
```
